## 1. 项目背景与目标

本项目的目标是开发一个面向 **四川血流成河麻将** 的本地视觉识别与出牌教学系统。

系统通过本地屏幕截图和 YOLO 目标检测自动识别当前牌局信息，包括自己的手牌、定缺、弃牌、碰杠、操作按钮等，并基于四川血流成河规则给出出牌、碰、杠、胡的教学建议。

最终系统不是简单的“算牌器”，而是一个完整的：

```text
本地截图采集 → YOLO 识别 → 牌局状态重建 → 血流规则引擎 → 策略决策 → 实时提示 → 对局日志 → 复盘训练
```

## 4. 为什么当前方案是合适的

### 4.1 视觉识别层：YOLO 是合理路线

麻将牌识别属于典型的目标检测任务。YOLO 适合实时检测、部署简单，并且支持自定义数据集训练。Ultralytics 官方文档支持通过 `data.yaml` 配置自定义数据集并用 `yolo detect train` 训练目标检测模型。

本项目采用：

```text
mss 截屏
+ OpenCV ROI 裁剪
+ YOLO 目标检测
+ 规则校验
+ 连续帧稳定确认
```

这是合适的工程路线。

### 4.2 数据层：本地截图是主数据源，网上数据是辅助源

网上有麻将目标检测数据集，可以用于冷启动、预训练或参考标注格式。例如 Roboflow Universe 上存在多个麻将目标检测数据集，包括数千张麻将图片的数据集。

但这些公开数据集通常不完全匹配你的真实游戏客户端、分辨率、牌皮肤、阴影、UI 风格，因此不能直接作为最终上线模型的主数据源。

正确的数据策略是：

```text
公开数据集：冷启动 / 预训练 / 参考格式
合成数据：补齐类别 / 平衡样本 / 数据增强
本地截图：最终精度来源
hard cases：持续提升识别鲁棒性
```

### 4.3 策略层：先规则与搜索，不直接强化学习

麻将是多人不完全信息博弈。日麻领域已有 Suphx 这类深度强化学习麻将 AI，但这种路线需要大量牌谱、复杂训练体系和较高算力成本。

四川血流成河规则相对日麻简单，但也有定缺、胡后继续打、查叫、查花猪、杠收益、连续胡等特殊机制。直接做端到端强化学习不适合作为第一阶段。

正确策略路线应为：

```text
基础规则牌效
→ 番型评分
→ 剩余牌估计
→ 防守风险
→ 蒙特卡洛模拟
→ 后续再考虑学习模型
```

## 7. 数据源规划

### 7.1 主数据源：本地自动截图

本地截图是最终模型精度的核心来源。

采集流程：

```text
打开麻将客户端
→ 启动采集程序
→ 每 300ms / 500ms / 1s 截图
→ 裁剪 ROI
→ 保存全屏截图和手牌区域截图
→ 记录时间戳、窗口大小、ROI 配置
```

目录建议：

```text
data/
├── raw/
│   ├── fullscreen/
│   ├── my_hand/
│   ├── discards/
│   ├── melds/
│   └── buttons/
│
├── labeled/
│   └── yolo_tiles/
│
├── datasets/
│   └── xueliu_tiles_v1/
│
└── hard_cases/
    ├── mis_detected/
    ├── low_confidence/
    └── occluded/
```

### 7.2 辅助数据源：公开麻将数据集

公开数据集用途：

```text
1. 冷启动模型；
2. 预训练；
3. 参考类别设计；
4. 参考 YOLO 标注格式；
5. 减少从 0 标注的成本。
```

注意：公开数据集不能直接替代本地截图数据。

### 7.3 辅助数据源：合成数据

合成数据用于快速补齐 27 类麻将牌。

生成方式：

```text
收集 27 种牌素材
→ 随机生成 13/14 张手牌排列
→ 加入背景、缩放、亮度、阴影、模糊
→ 自动生成 YOLO 标注框
→ 加入训练集
```

合成数据的作用：

```text
1. 平衡类别；
2. 补齐少见牌；
3. 提升初版模型鲁棒性；
4. 降低人工标注成本。
```

### 7.4 策略数据源：对局日志

每一手都保存：

```json
{
  "timestamp": "2026-06-25 21:30:00",
  "hand": ["1W", "2W", "3W", "5T"],
  "missing_suit": "T",
  "recommended_discard": "2T",
  "actual_discard": "5T",
  "shanten": 2,
  "ukeire": 14,
  "result": "unknown",
  "score_delta": 0
}
```

长期积累后，这些日志可以用于：

```text
1. 复盘；
2. 策略调参；
3. 评估系统推荐质量；
4. 后续训练学习模型。
```

## 11. 识别结果校验规则

YOLO 输出不能直接相信，必须经过麻将规则校验。

### 11.1 手牌数量校验

```text
有效手牌数量只能是 13 或 14 张。
```

如果识别到 12、15 或更多，当前帧判定为无效。

### 11.2 单牌数量校验

```text
同一种牌最多 4 张。
```

如果识别到 5 张 `3W`，当前帧判定为无效。

### 11.3 置信度校验

```text
低于阈值的检测框丢弃。
```

建议初始阈值：

```text
conf >= 0.60
```

### 11.4 重叠框校验

对重叠过高的框执行 NMS，只保留置信度最高的框。

### 11.5 排序规则

手牌区域识别完成后，按 x 坐标从左到右排序。

### 11.6 连续帧确认

```text
连续 2~3 帧识别结果一致，才确认当前手牌。
```

这样可以降低动画、遮挡、偶然误检带来的错误提示。

## 12. 四川血流规则引擎规划

### 12.1 基础规则

```text
1. 只有万、筒、条三门；
2. 每门 1~9；
3. 每种牌 4 张；
4. 无字牌；
5. 必须定缺；
6. 有缺门牌时必须优先打缺门；
7. 胡后继续打，直到牌墙结束；
8. 支持碰、杠、胡；
9. 支持查叫、查花猪；
10. 支持常见血流番型。
```

### 12.2 第一版支持番型

```text
1. 普通胡：4 面子 + 1 将；
2. 七对；
3. 碰碰胡；
4. 清一色。
```

### 12.3 后续支持番型

```text
1. 龙七对；
2. 金钩钓；
3. 杠上花；
4. 抢杠胡；
5. 根；
6. 海底胡；
7. 平胡规则差异；
8. 平台自定义番型。
```

## 13. 决策引擎规划

### 13.1 第一阶段：基础牌效

第一阶段不要追求“最优 AI”，先做稳定可靠的基础牌效。

基础评分：

```text
总分 =
  定缺优先分
+ 向听速度分
+ 有效进张分
+ 牌型结构分
+ 番型潜力分
- 破坏结构惩罚
```

决策规则：

```text
如果手里还有定缺门牌：
    候选出牌只从定缺门里选择；
    优先打孤张、幺九、结构价值低的缺门牌。
否则：
    枚举所有可打牌；
    计算打出后的向听数；
    计算有效进张；
    保留两面、复合搭子和高番潜力；
    推荐综合分最高的出牌。
```

### 13.2 第二阶段：操作建议

增加：

```text
1. 是否碰；
2. 是否杠；
3. 是否胡；
4. 是否过。
```

### 13.3 第三阶段：全桌信息决策

加入：

```text
1. 剩余牌估计；
2. 实际有效进张枚数；
3. 对手副露风险；
4. 哪些牌已绝张；
5. 哪些牌危险；
6. 是否应防守。
```

### 13.4 第四阶段：蒙特卡洛模拟

对每个候选出牌模拟后续牌局：

```text
打 9条 → 模拟 2000 局 → 平均收益 +2.4
打 1万 → 模拟 2000 局 → 平均收益 +1.1
打 6筒 → 模拟 2000 局 → 平均收益 -0.6
```

最终推荐期望收益最高的打法。

## 16. 第一阶段 Codex 开发提示词

```text
我要开发一个四川血流成河麻将本地视觉识别与出牌教学系统，项目名 xueliu-mahjong-ai。

最终目标：
这是一个本地运行的桌面辅助训练/复盘系统。它通过屏幕截图和 YOLO 目标检测自动识别麻将牌局，然后根据四川血流成河规则给出出牌、碰、杠、胡的教学建议，并保存对局日志用于复盘。

重要边界：
1. 不做自动点击；
2. 不读取游戏进程内存；
3. 不抓包；
4. 不绕过反作弊；
5. 只基于本地屏幕截图做图像识别；
6. 用于训练、复盘、单机练习和自用教学提示。

最终成品功能：
1. 自动识别麻将窗口或手动选择屏幕区域；
2. 实时截图；
3. YOLO 识别我的手牌；
4. 后续扩展识别弃牌区、碰杠区、定缺图标、操作按钮；
5. 重建 GameState；
6. 支持四川血流成河规则；
7. 自动推荐出牌；

10. 保存整局日志；
11. 生成复盘报告；
12. 支持 hard case 采集和模型持续训练。

第一阶段只做：
截图采集 + ROI 标定 + YOLO 数据集准备 + 我的手牌识别。

第一阶段具体要求：
1. Python 3.11+；
```

当前最合适的执行顺序是：

```text
第一优先级：外部基准验证
第二优先级：本地截图采集
第三优先级：YOLO 手牌识别稳定
第四优先级：基础出牌算法
第五优先级：实时悬浮提示
第六优先级：全桌状态识别
第七优先级：高级血流策略
第八优先级：蒙特卡洛模拟
第九优先级：复盘训练系统
```

第一阶段不要直接写复杂“最优麻将 AI”。

第一阶段真正要完成的是：

```text
真实屏幕数据 → ROI 裁剪 → YOLO 识别 → 规则校验 → 稳定输出手牌
```

只要这条链路跑通，后面的策略、复盘、蒙特卡洛都可以稳步叠加。
