# 血流麻将 AI 辅助项目现状梳理

日期：2026-07-05

## 1. 当前项目目标

本项目目标是做一个本地运行的血流麻将辅助系统：

- 从电脑屏幕或投屏窗口实时截图。
- 使用 YOLO 识别麻将牌面。
- 自动区分手牌、碰杠区、弃牌区、中间牌桌区域。
- 结合血流麻将规则计算牌型、向听数、有效牌和推荐出牌。
- 提供一个可点击启动的 UI 程序。
- 支持用视频回放做模型和区域逻辑测试。

当前已经进入“模型基本可用，主要问题转向实时区域归类和策略稳定性”的阶段。

## 2. 模型训练现状

当前主模型：

```text
models/yolo/xueliu_final325_clean_v1_0703.pt
```

该模型基于 clean start 后的人工标注数据训练，不再依赖早期未审核的 Roboflow/伪标注数据。

保留的训练阶段：

| 阶段 | 模型 | 数据来源 | 说明 |
| --- | --- | --- | --- |
| manual39 | `models/yolo/xueliu_manual39_clean_v1.pt` | 人工标注 39 张 | 第一轮干净人工样本 |
| manual113 | `models/yolo/xueliu_manual113_clean_v2_0702.pt` | 人工核对到 113 张 | 第二轮 |
| manual222 | `models/yolo/xueliu_manual222_clean_v3_0703.pt` | 人工核对到 222 张 | 第三轮 |
| final325 | `models/yolo/xueliu_final325_clean_v1_0703.pt` | 325 张全部人工标注 | 当前主模型 |

训练指标如下，来自 YOLO `results.csv` 最后一轮：

| 训练 run | epoch | precision | recall | mAP50 | mAP50-95 |
| --- | ---: | ---: | ---: | ---: | ---: |
| `final325_clean_v1_0703_high_precision` | 163 | 0.95806 | 0.95960 | 0.97367 | 0.82143 |
| `manual222_clean_v3_0703_high_precision` | 133 | 0.96498 | 0.96775 | 0.97195 | 0.80486 |
| `manual113_clean_v2_0702_high_precision` | 152 | 0.94813 | 0.94698 | 0.95875 | 0.76647 |
| `manual39_clean_v1_high_precision` | 160 | 0.83112 | 0.80333 | 0.88248 | 0.66348 |

结论：

- 单张牌识别能力已经明显提升，最终模型 mAP50 约 0.974。
- 错误主要不再是“模型完全不认识牌”，而是实时画面里某些牌跨区域、临时过渡帧、遮挡、发牌动画导致区域归属不稳定。
- 当前模型仍缺少更多“真实投屏连续牌局”的样本，尤其是发牌、摸牌、碰杠后手牌区收缩、弃牌区堆叠、不同玩家视角的碰杠牌。

## 3. clean start 前的问题

早期训练遇到的主要问题：

1. 外部数据集风格不一致
   - Roboflow、Riichi、MajSoul、普通麻将牌图等数据源与当前真实游戏 UI 差异大。
   - 有些图片是实物麻将，有些是日麻，有些标签命名和我们的血流麻将不完全一致。

2. 未审核数据污染模型
   - 曾经用模型自动预标注后继续训练，出现 `2筒 -> 4筒`、躺着的牌识别错等问题。
   - 高置信错标进入训练集后，模型会越来越自信地学错。

3. 多角度和密集牌表现不稳定
   - 左右旋转 90 度、180 度、碰杠堆叠、弃牌区密集牌容易错分或漏检。

4. 回测图遮挡人工判断
   - 早期回测图叠加了大量置信度、来源、说明文字，影响人工审查。
   - 后续已改成左上角标签为主。

clean start 后的原则已经改为：

- 训练集只使用人工确认过的标注。
- 不再把未经核对的自动标注直接作为真值训练。
- 外部数据集只作为参考，不再作为当前主模型训练依据。

## 5. 当前遇到的问题

### 5.1 区域归属仍是最大问题

真实牌局中手牌区和碰杠区是动态变化的：

- 没有碰杠时，底部几乎全是手牌。
- 碰一组后，碰杠区增大，手牌区缩短。
- 碰两组、三组后，手牌数量继续减少。
- 摸进的新牌可能短暂离主手牌列较远。

目前靠几何逻辑动态分区，已经能处理大多数场景，但仍会出现：

- 右侧碰牌被误并入手牌。
- 手牌尾部单张和碰杠区边界混淆。
- 发牌/选牌动画过程中手牌数量不足，被判为异常。

### 5.2 预览与 UI 数值可能短暂不一致

预览窗口显示的是当前帧识别框，右侧 UI 显示的是稳定后的手牌和策略结果。稳定器会延迟确认，避免单帧误识别，所以两边可能短暂不同。

这个行为是有意设计，但 UI 上后续需要更明确地区分：

- 当前帧检测结果
- 稳定后结果
- 策略使用结果

### 5.3 牌局阶段判断还不完整

当前只是用手牌数量、碰杠区数量和异常诊断来判断是否能推荐，还没有完整识别：

- 发牌中
- 换三张
- 定缺
- 选牌中
- 出牌中
- 碰/杠/胡操作提示
- 结算页

这些阶段如果误进入策略，会出现无意义推荐或区域异常提示。

### 5.4 策略层还是初级版本

目前策略主要基于：

- 定缺优先
- 普通牌型向听
- 七对向听
- 有效牌数量
- 已见牌数量

尚未完整加入：

- 血流麻将换三张策略
- 定缺自动判断
- 碰杠收益判断
- 番型/倍数精算
- 对局阶段风险控制
- 根据对手弃牌推断安全牌
- 多轮模拟或期望收益估计

### 5.5 数据仍然偏少

325 张人工标注对当前 UI 已经有效，但覆盖还不够。后续需要更多真实投屏视频样本，尤其是：

- 发牌过程
- 换三张过程
- 定缺过程
- 摸进单张
- 自己碰/杠后手牌区变化
- 上家/对家/下家碰杠
- 四家弃牌区逐渐堆叠
- 胡牌/结算/弹窗遮挡

## 9. 下一步建议

下一步建议按这个顺序继续：

1. 准备新的真实投屏训练视频。
2. 用 `video-replay` 或单独抽帧脚本抽取关键帧。
3. 先人工筛掉结算页、加载页、遮挡严重页面。
4. 在 LabelU/CVAT 里人工标注，不直接使用模型错标作为真值。
5. 新增一批专门的“区域训练/测试样本”，尤其是碰杠后手牌区变化。
6. 训练新模型后，用黄金测试集和视频回放做固定回归测试。
7. 继续增强实时区域状态机和牌局阶段识别。

短期最重要的是补齐“区域状态”样本，而不是盲目增加普通手牌样本。
