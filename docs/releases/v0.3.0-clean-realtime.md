# v0.3.0 clean realtime development notes

Date: 2026-07-05

## What changed

- Runtime default model is `models/yolo/xueliu_final325_plus_longjing39_plus83_clean_v1_0709.pt`.
- `models/yolo/training_rounds.yaml` now only describes the clean-start line:
  `manual39 -> manual113 -> manual222 -> final325`.
- Added `python -m xueliu_ai replay-test` for curated regression frames.
- Added game phase inference and recommendation gating:
  - loading
  - dealing
  - choose missing suit
  - waiting
  - my turn
  - settlement
- Realtime UI now shows phase and recommendation gate status.
- Added my-area analysis for hand count, drawn tile, meld tile count, and legality.
- Added strategy helpers for:
  - missing suit recommendation
  - exchange-three recommendation
  - peng/kong advice
  - basic fan pattern summary
- Added `scripts/package_runtime.py` to create sanitized runtime zip files.

## Current model

Main model: `models/yolo/xueliu_final325_plus_longjing39_plus83_clean_v1_0709.pt`

Training data: 325 manually labeled clean-start images.

Known limitation: current model still needs more hard cases around dynamic bottom hand/meld boundaries, split drawn tile, and side meld/discard ownership. These should be collected and manually verified before training any successor model.

## Hard-case model switch rule

Do not switch the runtime model only because YOLO mAP improves.

A future model can become the runtime default only if:

1. It is trained from manually verified labels or fully reviewed prelabels.
2. It passes `replay-test` without regression against final325.
3. It improves the hard-case tags that motivated the new round.
4. `training_rounds.yaml` is updated with rollback information.
