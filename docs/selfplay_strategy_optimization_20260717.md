# 血流麻将自对弈策略优化

日期：2026-07-17

## 目标变更

项目策略目标由“教学和解释人类牌理”调整为：

> 在给定血流规则和当前可观测信息下，最大化整局平均净收益，并以最终胜率作为辅助指标。

人类牌谱不是必要前提。当前路线采用规则模拟、概率计算、自对弈和参数搜索。人类数据未来只用于校准真实对手行为分布，不作为策略上限。

## 已实现

- 108 张序数牌牌墙。
- 四人轮转和固定随机种子。
- 初始发牌、换三张和定缺。
- 摸牌、合法弃牌和定缺强制清理。
- 碰牌和碰后出牌。
- 暗杠、补牌和刮风下雨基础分转移。
- 自摸、点炮和一炮多响。
- 血流模式下胡牌后继续对局。
- 零和分数转移。
- 每步牌数守恒检查。
- 快速规则代理、价值代理和随机对照代理。
- 轮换座位的批量锦标赛。
- 多进程执行入口。
- 策略参数随机搜索。
- 冠军参数和完整实验清单输出。

## 命令

运行基础锦标赛：

```powershell
python -m xueliu_ai selfplay --games 1000 --seed 20260717 --workers 4
```

搜索快速代理参数：

```powershell
python -m xueliu_ai selfplay-optimize `
  --candidates 24 `
  --games 200 `
  --seed 20260717 `
  --workers 4 `
  --output data/selfplay/latest_optimization.json
```

## 优化指标

主指标：

- `average_score`：每局平均净收益。

辅助指标：

- `win_rate`：最终分数并列最高的比例。
- `average_hu_count`：每局平均胡牌次数。
- 对不同对手组合和不同座位的稳定性。

不能只优化胡牌率。更高的胡牌次数可能伴随更大的点炮损失，最终仍以净收益为准。

## 实验纪律

- 所有实验必须记录随机种子。
- 候选和基线必须轮换座位。
- 同一轮候选尽量使用相同种子集合，降低牌运噪声。
- 冠军策略必须在未参与调参的新种子上复验。
- 小样本结果只用于筛选，不能直接宣布更优。
- 规则版本变化后，旧实验不能与新实验直接比较。

## 当前规则边界

模拟器已经具备闭环运行能力，但以下项目必须按实际客户端继续校准：

- 每种番型的准确番数和叠加规则。
- 封顶方式。
- 自摸和点炮支付方式。
- 明杠、暗杠、补杠和退税细节。
- 查花猪和查大叫的最终结算。
- 胡后具体换牌、摸打和再次胡牌规则。
- 流局和牌墙结束条件。

这些差异都应通过规则配置和回归案例修正，不能混入代理策略代码。

## 后续演进

1. 完成实际客户端结算规则校准。
2. 增加补杠、抢杠胡和退税完整状态机。
3. 增加对手隐藏手牌的信念状态。
4. 用自对弈结果训练动作价值模型。
5. 以价值模型作为蒙特卡洛搜索的叶节点估值。
6. 建立冠军池，避免新策略只针对单一基线过拟合。
7. 在真实对局日志上检查模拟策略与实际收益的分布偏差。
